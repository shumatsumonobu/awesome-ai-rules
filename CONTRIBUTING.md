# Contributing to Awesome AI Rules

Thank you for helping grow this collection!

## Adding an Example

1. **Verify the rule file exists** -- link to the actual file (e.g., `github.com/org/repo/blob/main/CLAUDE.md`), not just the repo root
2. **Check star count** -- approximate is fine (e.g., "12k")
3. **Write a clear description** -- one sentence on what makes it a *good example*, not just what the project does
4. **Place it correctly** -- right format section, ordered by star count (highest first)

## PR Format

```
| [org/repo](https://github.com/org/repo) | 12k | Language | One sentence on what makes it good |
```

## Quality Bar

We include examples that teach something -- a novel pattern, a well-structured file, or a creative use of the format. Generic or boilerplate rule files don't make the cut.

## What We Don't Include

- Rule file generators or templates (unless they're notable enough to be in "See also")
- Private or archived repositories
- Repositories with fewer than 100 stars (unless the rule file itself is exceptionally well-written)

## Reporting Issues

Found a broken link or incorrect info? Open an issue or PR with the fix.

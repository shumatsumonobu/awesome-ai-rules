# packages/api

Drizzle ORM schema, services, and database queries. This file extends the root CLAUDE.md -- Claude Code reads the root first, then merges this file when working inside `packages/api/`.

## What This File Adds (not in root)

- Use Drizzle's query builder, not raw SQL.
- Define tables in `src/schema/`, one file per table.
- Services return plain objects, never Drizzle row types -- map to shared types from `packages/shared/`.
- Throw domain-specific errors (`NotFoundError`, `ValidationError`), not raw database errors.

## What This File Overrides (different from root)

- **Commands**: Use `pnpm --filter api test/db:generate/db:migrate` here, not root-level commands.
- **No frontend code**: Never import from `react`, `next`, or any UI library in this package. If you see a React import here, it's a bug.

## Isolation Rule

The web package never imports `drizzle-orm` directly. All database access goes through this package's service layer. If someone asks to "query the database from a component," create a service method here and expose it via a Server Action in `packages/web/`.

## Directory Structure

```
src/
  schema/       # Drizzle table definitions (one file per table)
  services/     # Business logic (called by web's Server Actions)
  queries/      # Reusable query builders
  migrations/   # Generated by drizzle-kit (do not edit by hand)
```
